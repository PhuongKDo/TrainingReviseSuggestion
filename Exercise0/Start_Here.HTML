Links, Advice, Warnings

Glossary

  1. "Bean"
    • A .cfc file with getter and setter methods populated by a service.
    • ColdFusion is based on Java, so the bean refered to in FW/1 is like a JavaBean
    • Note: a "Bean" is called a domain at Mind Over Data (MOD).
  2. "Gateway"
    • A .cfm file with cfquery and cfqueryparam tags.
    • cfquery param tags are used to run SQL queries.
    • Note: framework 1 does not provide documentation for writing Gateways.
  3. "Service"
    • A .cfc file with buisness logic methods for modifying and collecting into beans data from the gateway.
  4. "Domain"
    • A .cfc file with the object classes created for the application.
  5. "API"
    • Stands for Application Programming Interface.
    • APIs allow developers to transfer data from outside the scope of the application.
    • For example, an API can provide results from database query to a remote data source.
    • I use Postman (UI) and cURL (script) to interact with Web APIs.
  6. "Dependency injection"
    • Allows access variables like "session" or "rc" in a view or controller
      without declaring them (DI/1 injects these variables).
  7. "buildURL"
    • Urls are structured in Framework 1 to isolate the action accomplished by a page.
    • For example, a page listing products would be /index.cfm?action=product.list
  8. "CTE"
    • A decent Wikipedia article on CTE
    • CTE stands for Common Table Expression.
  9. "SPA"
    • A decent Wikipedia article on SPA
    • SPA stands for Single Page {web} Application.
  10. "SOAP"
    • SOAP (rember, this is HTTP/S using XML)
    • SOAP stands for Simple Object Access Protocol.
  11. "REST"
    • REST (rember, this is HTTP/S using XML, HTML or JSON for CRUD HTTP opperations)
    • REST stands for Representational State Transfer.

Advice

  1. Debug Framework One Applications (esspecially DI/1 and FW/1).
    • Use ‹cfdump var=''› ‹/cfdump › and ‹cfabort› ‹/cfabort › tags in .cfm files to figure out what values are assigned to each structure in rc or session
  2. Use libraries and existing code as much as possible.
    • DataTables, Bootstrap 4, JQuery and even your own code from a prior project can be used to eliminate redundancy.
  3. Don't overlook the webConfig file and Application.cfc as potencial sources of errors.
    • Make sure that your environmnet is set up correctly.
    • Check your mappings to verify they match all your required file types.
    • Verify that your url building functions work as expected.
